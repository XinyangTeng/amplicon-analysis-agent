# Architecture

The system deliberately separates biological judgment, deterministic execution, and language-model interpretation.

```text
User and project files
        |
        v
LLM + expert Skill
  - parse the biological question and experiment design
  - organize required inputs
  - select eligible analyses and explain limitations
        |
        v
MCP Server
  - validate tool schemas and workspace paths
  - inspect inputs and create an immutable Analysis Contract
  - enforce file hashes and one-time approval
  - orchestrate execution, validation, status, and audit records
        |
        v
Deterministic executors
  - baseline R analysis
  - team EMO functions in batch-isolated workspaces
        |
        v
Deterministic validation and report builder
  - validate statistical and file invariants
  - recursively scan all output files
  - group figures by experiment batch and result section
  - write report.html, report_data.json, and artifact_manifest.json
        |
        v
LLM interpretation
  - read validated report_data.json through get_report_context
  - separate facts, statistics, interpretation, and hypotheses
```

## Why the report is not generated by the LLM

File discovery, image placement, result indexing, and execution-state checks are deterministic tasks. The report builder therefore scans the run directory after all requested modules finish. A newly added module can contribute PNG/JPG/SVG figures, result tables, PDFs, and logs without changing the HTML template.

`report_data.json` is the language-model boundary. It contains the contract summary, validation results, QC summary, statistical result JSON, EMO execution states, artifact inventory, and interpretation policies. The model reads it only after `validate_amplicon_results` passes.

The HTML is a reproducible presentation of generated files, not an AI-written scientific narrative. Biological interpretation remains conversational and can be revised without rerunning the analysis or changing the primary report.
